## Git運用ルール

### 基本方針

- コードを変更するたびに、変更内容をコミットしてGitHubにプッシュする
- コミットは論理的な単位でまとめ、意味のあるメッセージを付ける
- `main` ブランチへの直接の force push は禁止

### コミット手順

コードを変更した後は必ず以下の手順を実行する：

1. 変更内容を確認する（`git status` / `git diff`）
2. 関連ファイルをステージングする（`git add <files>`）
3. コミットメッセージを付けてコミットする
4. GitHubにプッシュする（`git push`）

### コミットメッセージの書き方

- 日本語または英語で記述する
- 変更の「何を」ではなく「なぜ」を重視する
- 例：`タスクの並び替え機能を追加` / `fix: 完了タスクが削除できないバグを修正`

### ブランチ戦略

- `main`: 本番相当のブランチ
- 新機能や大きな変更は feature ブランチを切って作業し、PR経由でマージする
- 小さな修正は `main` に直接コミット・プッシュして構わない
